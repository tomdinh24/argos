## C1 root-cause analysis

Two distinct issues, only one is a real Brief defect:

### Real bug: structured flag override

The narrative for CLM-015 says "represented by T. Olsen, Esq." but
the loss_facts_hint explicitly says `represented: False` (claim
`rep_flag=False`). T. Olsen's name appears in correspondence
documents on file, so the LLM inferred representation from doc
signatures rather than trusting the structured flag.

**Fix:** add a rule to `narrative.py` SYSTEM_PROMPT — when
loss_facts_hint contains `represented: False`, do not describe the
claimant as represented, even when attorney correspondence is on
file. (The attorney may be writing on someone else's behalf, or the
file flag may lag the doc; either way the structured flag is
authoritative.) The same rule applies to litigation_flag.

Same pattern likely caused the "coverage under ROR pending" call on
CLM-007 — the narrative invented "ROR" when status was just
"pending". Fix: tighten prompt to say "use coverage_status verbatim".

### Judge non-determinism

The auto-run counted 2/1/3 hallucinations across the three claims; a
manual re-run on CLM-015 alone counted 1. Same prompt, same model,
different verdicts. This is expected — the judge is itself an LLM —
but it means the strict "0 hallucinations" pass bar was too tight to
be meaningful with a single run. Either:

- (a) use a deterministic narrative-fact extractor instead of an LLM
  judge (regex out dollar amounts/dates/names + substring-check
  against source corpus), or
- (b) run the judge N=3 times and require the *median* count to be
  zero.

This is an eval method issue, not a Brief issue.

## Substantive things that worked

- Both LLM calls landed in 1 attempt for all 3 claims (no retries).
- Narratives quote source docs accurately for medical facts (C5-C6
  herniation, $85K–$120K), legal facts (cooperative-defense
  paraphrase, tender denial), and structured facts (open date,
  severity, coverage type, claim_id, request_id).
- Status snapshot, coverage-result consumption, and gap-recall logic
  all behave correctly.

## Proposed run-3 changes

Two changes; both require a new lock revision (r2):

1. **Brief change:** tighten `narrative.py` SYSTEM_PROMPT with two
   new rules — (a) "Use `represented:` and `litigation_flag:` from
   loss_facts_hint verbatim; do not infer representation or
   litigation from attorney correspondence in documents."
   (b) "Use `coverage_status:` verbatim from loss_facts_hint; do
   not paraphrase 'pending' as 'ROR' or 'denied' as 'tender
   declined'."
2. **Eval change:** either swap the LLM judge for a deterministic
   fact extractor, or run the judge N=3 and require median = 0.

Until those changes are made, treat the substantive verdict as
**Brief is functionally correct with a known narrative discipline
gap** — the deterministic system behaves; the narrative drifts on
two specific patterns that prompt engineering can fix.
